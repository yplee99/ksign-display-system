# ksign-display-system — 프로젝트 컨텍스트

## 프로젝트 개요

BD Rowa™ Vmotion 시스템을 참조하여 개발하는 **AI 기반 디지털 사이니지 및 3D 가상 진열장 애플리케이션**이다.
약국 등 소매 환경에서 터치스크린 디스플레이를 통해 제품을 3D 가상 선반에 진열하고, 고객이 직접 탐색·조회·구매 요청할 수 있는 시스템을 목표로 한다.

## 참조 제품: BD Rowa™ Vmotion

본 프로젝트는 아래 제품의 구조와 기능을 참고한다.

### 하드웨어 구성
- **디스플레이**: 22"~75" 터치/비터치 스크린 (디지털 OTC 선반, 쇼핑 윈도우)
- **서버**: Intel NUC i3 (외장형) 또는 SoC (화면 내장형)
- **운영 시간**: 대부분의 모델 24/7 운영
- **설치 방식**: 벽걸이, 테이블 매립, 폴 설치, 레벨링 풋

### 소프트웨어 구성 (Vmotion Cloud)
클라우드 기반 웹 포털로 콘텐츠를 관리한다. 로컬에 설치되지 않고 웹 애플리케이션으로만 동작한다.

| 메뉴 | 역할 |
|------|------|
| Dashboard | 연결된 화면 수, 활성 플레이리스트/스케줄 현황 |
| Planograms | 가상 선반 생성·편집 (배경, 헤더, 제품 배치) |
| Playlists | 플래노그램을 묶어 순서대로 재생하는 목록 |
| Schedule | 날짜·시간별 플레이리스트 활성화 일정 |
| Pulldown Menus | 터치 화면에서 위→아래 스와이프 시 보이는 메뉴 |
| Media | 이미지·비디오·PDF 업로드 및 관리 |
| Products | 가격, 할인, 재고, 대안 제품, 추천 제품 관리 |
| Vmotion Settings | 가격 표시, 팝업, 바코드, 화면 자동 꺼짐 등 설정 |
| My Organizations | 다수 약국 조직 간 전환 관리 |

## 핵심 용어

- **플래노그램(Planogram)**: 배경 이미지·헤더·비디오를 결합하여 제품 패키지를 자유롭게 배치한 가상 선반 화면
- **플레이리스트(Playlist)**: 여러 플래노그램을 묶어 순서대로 화면에 표시하는 목록 (화면당 최대 50개)
- **스케줄(Schedule)**: 플레이리스트를 특정 날짜·시간에 자동 활성화하는 예약 설정
- **풀다운 메뉴(Pulldown Menu)**: 고객이 화면 상단에서 아래로 스와이프하면 나타나는 플래노그램 선택 메뉴
- **NUC**: 화면에 연결하는 소형 외장 PC (Intel NUCi3, 250GB SSD, Windows 10)
- **SoC**: 화면 내장형 미디어 플레이어 (55" 모델에 사용)

## 화면 조작 방식

- 한 손가락만 사용
- 좌우 스와이프: 플래노그램 전환
- 위→아래 스와이프: 풀다운 메뉴 열기
- 아래→위 스와이프: 소프트웨어 제어 메뉴 (종료·잠금·회전)
- 제품 탭: 상세 팝업 (이름, 가격, 재고, 2D/3D 뷰, 비디오, 대안 제품, 조제 버튼)

### 재고 표시 기호
- ▲ : 20개 초과
- ▼ : 1~20개
- ● : 재고 없음

## 플래노그램 디자인 원칙

- 선반당 최대 4개 제품 (가시성 확보)
- 4~5개 선반 권장
- 같은 제품 스택: 가로 최대 4개, 세로 최대 3개
- 배경 이미지와 텍스트 간 충분한 명암 대비 필요
- 제품 간 여백, 선반 끝 여백 유지

## 미디어 업로드 규격

| 종류 | 해상도/형식 | 제한 |
|------|-----------|------|
| 배경·파노라마 이미지 | min 1585 × 1920 px | JPG, PNG, PDF |
| 헤더 이미지 | min 1585 × 300 px | JPG, PNG, PDF |
| 비디오 | Full HD | max 350MB, MP4 |

## 콘텐츠 업데이트 주기

Vmotion Cloud에서 변경 후 화면에 반영되기까지 약 **5분** 소요.
